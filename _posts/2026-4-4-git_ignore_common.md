`.gitignore` 用于告诉 Git 哪些文件或目录不需要被版本控制（即不提交到仓库）。正确配置 `.gitignore` 可以避免提交临时文件、编译产物、本地配置、依赖目录或大型数据文件等。

## 1. 基本语法规则

| 规则示例         | 含义                                                         |
| ---------------- | ------------------------------------------------------------ |
| `temp.txt`       | 忽略仓库中**任何位置**名为 `temp.txt` 的文件                 |
| `logs/`          | 忽略仓库中**任何位置**名为 `logs` 的**目录**（包括目录内所有内容） |
| `/build/`        | 只忽略**仓库根目录下**的 `build` 目录                        |
| `*.log`          | 忽略所有 `.log` 后缀的文件                                   |
| `!important.log` | 取反，不忽略 `important.log`（但需先忽略 `*.log`）           |
| `docs/*.txt`     | 忽略 `docs/` 目录下的 `.txt` 文件，但不影响 `docs/sub/` 下的 |
| `docs/**/*.txt`  | 忽略 `docs/` 下任意层级子目录中的 `.txt` 文件                |

> **注意**：`/` 开头的模式表示“从仓库根目录开始匹配”；末尾的 `/` 表示匹配目录（而非文件）。

## 2. 忽略特定目录的两种常见写法

假设您要忽略一个名为 `data/` 的目录：

### 写法一：只忽略根目录下的这个目录
```gitignore
/data/
```
- 只有当 `data` 目录直接位于仓库根目录下时才被忽略。
- 如果根目录下有同名**文件**（而非目录），不会被此规则匹配（因为末尾 `/` 限定为目录）。

### 写法二：忽略仓库中任何位置出现的同名目录
```gitignore
data/
```
- 无论 `data` 目录出现在根目录、子目录还是多级子目录中，都会被忽略。

> **建议**：对于项目依赖、数据集等通常放在根目录下的内容，使用写法一更精确；对于可能散落在各处的临时目录（如 `__pycache__/`、`node_modules/`），使用写法二更方便。

## 3. 路径模式中的通配符

| 通配符  | 说明                             | 示例                                                |
| ------- | -------------------------------- | --------------------------------------------------- |
| `*`     | 匹配零个或多个字符（不包括 `/`） | `*.tmp` 忽略所有 `.tmp` 文件                        |
| `**`    | 匹配任意级目录                   | `data/**/raw/` 忽略所有 `raw` 子目录                |
| `?`     | 匹配单个字符                     | `image?.png` 匹配 `image1.png` 不匹配 `image10.png` |
| `[abc]` | 匹配括号内的任一字符             | `[0-9].txt` 匹配 `1.txt`                            |
| `\`     | 转义特殊字符                     | `\!secret.txt` 忽略名为 `!secret.txt` 的文件        |

## 4. 已经跟踪的文件怎么办？

**重要**：`.gitignore` 只对**尚未被 Git 跟踪**的文件生效。如果某个文件/目录已经被提交过（即 `git add` 并 commit 过），即使后来写入 `.gitignore`，Git 仍会继续跟踪它的变化。

**解决方案**：先从 Git 索引（暂存区）中移除该文件/目录，但保留本地文件。

### 移除单个文件
```bash
git rm --cached large_file.dat
```

### 移除整个目录（递归）
```bash
git rm -r --cached data/
```

执行后，这些文件/目录会从仓库中删除（下次 commit 生效），但本地磁盘上的文件仍然存在。此时 `.gitignore` 规则就会阻止它们被重新添加。

### 一次性处理多个已跟踪目录
```bash
git rm -r --cached data/ logs/ temp/
```

## 5. 实战示例：完整的 .gitignore 片段

```gitignore
# 忽略根目录下的数据、日志、临时目录
/data/
/logs/
/temp/

# 忽略 Python 编译文件
__pycache__/
*.py[cod]

# 忽略 IDE 配置（如 VSCode、PyCharm）
.vscode/
.idea/

# 忽略环境/依赖目录（如果放在根目录）
/venv/
/env/
/node_modules/

# 忽略日志和临时文件
*.log
*.tmp
.DS_Store
```

## 6. 调试技巧

### 查看当前哪些文件被 Git 忽略
```bash
git status --ignored
```

### 检查某条规则是否匹配了特定文件
```bash
git check-ignore -v path/to/file
```
输出会显示是哪条 `.gitignore` 规则（以及来自哪个文件）导致该文件被忽略。

### 测试不同路径模式的匹配效果
可以在 `.gitignore` 中临时添加规则，然后用 `git add --dry-run` 预览哪些文件会被添加：
```bash
git add -A --dry-run
```
未被添加的文件中，被忽略的会显示为“ignored”。

## 7. 大小写敏感性

- `.gitignore` 中的路径模式**区分大小写**（除非操作系统或 Git 配置了大小写不敏感）。
- 目录名 `Data` 和 `data` 会被视为不同，必须与实际大小写完全一致。

## 总结

| 需求                     | 写法                                               |
| ------------------------ | -------------------------------------------------- |
| 忽略根目录下的 `data/`   | `/data/`                                           |
| 忽略任意位置的 `data/`   | `data/`                                            |
| 忽略已跟踪的目录         | 先 `git rm -r --cached <dir>`，再加入 `.gitignore` |
| 忽略所有 `.tmp` 文件     | `*.tmp`                                            |
| 保留被全局忽略的特定文件 | `!keep_this.tmp`                                   |

编写 `.gitignore` 时，优先使用**根目录限定**（`/dir/`）避免误忽略，只对确需全局忽略的目录使用非限定写法。并记住：已经提交的文件需要先解除跟踪，`.gitignore` 才会接管。
